PISA 2012: Global Student Performance

View Analysis

The Problem

As part of the Udacity Data Analyst Nanodegree, this project investigates the PISA 2012 dataset. The goal was to identify which variables—ranging from gender and geography to teacher practices—most significantly impact the academic success of 15-year-old students globally.

The Data

Volume: 510,000 student records with 636 unique features.
Focus: Socio-economic status, school resources, and regional disparities (OECD vs. Non-OECD).
Download Raw Dataset

Methodology

  • Data Wrangling: Cleaning a massive 600+ column dataset using Pandas.
  • Multivariate Visualization: Using Seaborn and Matplotlib to find correlations between wealth and test scores.
  • Hypothesis Testing: Statistical validation of performance gaps between genders and regions.

Global Insights

Parental Influence: High parental education levels proved to be the strongest predictor of student success across all tested regions.

Digital Divide: Access to school computers improved scores, yet students in tech-equipped schools who *never* used them scored the lowest.

Regional Excellence: Students in East Asia and Western Europe significantly outperformed the global average in Mathematics.

Sprocket Central: Customer Growth Strategy

Open Interactive Dashboard

The Business Challenge

Sprocket Central Pty Ltd needed to identify which of their 1,000 new potential customers would yield the highest Customer Lifetime Value (CLV). This project transitions from descriptive analytics to predictive strategy, ensuring marketing budgets are spent only on high-conversion prospects.

Integrated ML & BI Methodology

  • RFM Segmentation: Engineered Recency, Frequency, and Monetary features to categorize existing customers into "Platinum," "Gold," and "At Risk" segments.
  • Predictive Scoring: Applied a propensity model to project purchase likelihood across 1,000 new prospects based on demographic and geographic indicators.
  • Advanced DAX Modeling: Developed complex measures for Gross Profit margins and dynamic ranking to visualize segment performance in real-time.

Data Architecture

Integration: Combined 3 disparate tables (Demographics, Addresses, and Transactions) into a unified Star Schema for optimized Power BI performance.

Strategic Outcomes

382

High-Value Targets: Identified a core group of prospects projected to drive 29,000+ unit sales.

56%

Regional Dominance: NSW remains the primary market, followed by VIC (26%) and QLD (21%).

The "Gold" Persona: Females aged 40-50 and Males in the 20-30/60-70 brackets showed the highest propensity for premium brand loyalty.

Top Brands: Solex (20%), Giant (17%).

Aviation Intelligence: Revenue & Operations Optimization

View Notebook & SQL

The Mission

This project delivers a 360-degree view of airline operations by merging financial ticket data with flight schedules. The objective was to identify revenue leakage, optimize hub performance, and predict operational bottlenecks using a hybrid SQL-Python workflow.

The Tech Stack & Methodology

  • Advanced SQL (SQLite): Leveraged Common Table Expressions (CTEs) and Window Functions (RANK, LEAD/LAG) to perform complex revenue attribution across multi-leg flights.
  • Python EDA: Utilized Pandas for data cleaning and Seaborn/Matplotlib for statistical distribution analysis and identifying correlation between aircraft types and fuel efficiency.
  • Predictive Analysis: Analyzed delay patterns to identify high-risk routes, enabling proactive scheduling adjustments.
  • Looker Studio: Engineered a dynamic executive dashboard for real-time monitoring of KPI trends.

Data Sources

Repository: Kaggle Airlines Dataset.
Architecture: Relational schema including ticket sales, flight schedules, delay codes, and airport geolocation data.

Strategic Insights

1.9B

Top Revenue Hub: Domodedovo Airport emerged as the primary financial driver, significantly outperforming Sheremetyevo (1.5B).

63%

Peak Seasonality: Identified July as the critical sales window, allowing for optimized dynamic pricing strategies.

>3 hrs

Operational Risk: Routes to Moscow and Kazan showed recurring long-tail delays, pinpointed through predictive pattern matching.

Data Infrastructure & ETL

Multi-Source Ingestion: API & Unstructured Data Pipeline

Engineering Architecture

Successfully engineered an ETL pipeline that synchronizes fragmented data from three distinct sources: Twitter’s REST API (Tweepy), cloud-hosted TSV files, and archival CSV datasets. This project demonstrates the ability to manage complex "Schema-on-Read" challenges by merging disparate data formats into a unified, analytics-ready master file.

Technical Implementation

  • Ingestion Layer: Developed a programmatic retriever using the Requests library and integrated Tweepy to handle authentication and rate-limited JSON metadata extraction.
  • Transformation Engine: Utilized Regular Expressions (RegEx) to parse unstructured tweet text and resolve tidiness issues across 5,000+ observations, specifically normalizing dog stage variables.
  • Quality Framework: Built a programmatic validation suite to assess data across four quality dimensions: Completeness, Validity, Accuracy, and Consistency.
  • Storage: Automated the final merge and export of cleaned data into a twitter_archive_master.csv storage file.

System Specs

API

Tweepy Integration: Handled JSON parsing to extract missing retweet and favorite metadata directly from Twitter.

ETL

Process: Implemented a rigorous "Assess-Clean-Test" workflow to improve both data quality and structural tidiness.
