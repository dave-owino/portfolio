The Problem

Identifying patterns in criminal activity is essential for evidence-based resource allocation and proactive policing. The Data

Source: New Zealand Police Department (via Kaggle).
Scope: Six months of reported vehicle theft incidents across New Zealand regions.
Attributes: Vehicle type, make, model, year, color, date stolen, and geographic region.

Methodology

  • SQL Joins & CTEs: Integrating six relational tables to connect theft events with detailed vehicle characteristics.
  • Exploratory Data Analysis (EDA): Identifying dominant vehicle types, makes, model years, and regional theft distributions.
  • Advanced Aggregations: Analyzing theft frequency by date, weekday, region, and vehicle attributes to surface hidden patterns.
  • Google Looker Studio: Translating SQL outputs into an interactive, executive-ready dashboard for decision-makers.

Key Insights

95%

of stolen vehicles were Standard models, while Luxury vehicles accounted for just 4%, indicating that opportunistic theft outweighs high-end targeting.

36%

of all vehicle thefts occurred in Auckland, followed by Canterbury (14.5%), highlighting clear regional concentration.

M/T

Thefts peaked on Mondays and Tuesdays, with the single highest spike recorded on April 4, 2022.

2003–08

Vehicles manufactured between 2003 and 2008 were most frequently stolen, suggesting older models with weaker security features are more vulnerable.

Most Targeted: Station wagons, sedans, and hatchbacks.
Top Makes: Toyota (15.7%), Trailer (11.9%), Nissan (10.6%).

PISA 2012: Global Student Performance

View Analysis

The Problem

As part of the Udacity Data Analyst Nanodegree, this project explores the PISA 2012 dataset to understand the factors that influence academic performance among 15-year-old students worldwide. The analysis investigates how gender, school choice, location, student attitudes, and teacher practices interact with socio-economic conditions to shape outcomes in mathematics, science, and literacy across OECD and non-OECD countries.

The Data

Source: OECD Programme for International Student Assessment (PISA).
Scale: ~510,000 students, 485,490 observations, and 636 features.
Coverage: Academic scores, socio-economic background, school resources, teacher-student relations, and technology access.
Download Raw Dataset

Methodology

  • Data Wrangling: Cleaning and restructuring a high-dimensional dataset (600+ variables) using Pandas for efficient analysis.
  • Exploratory & Multivariate Analysis: Leveraging Matplotlib and Seaborn to uncover relationships between gender, socio-economic status, and academic outcomes.
  • KPI-Driven Evaluation: Measuring the impact of teacher-student relations on gender-based performance differences.
  • Statistical Reasoning: Validating observed trends through distributional comparisons across OECD and non-OECD groups.

Global Insights

Parental Education: Higher parental education levels are strongly associated with improved student performance across mathematics, science, and literacy, with female students outperforming males at most education levels.

Technology Access: Access to desktop computers at school correlates with higher scores; however, students who never used available computers consistently performed worse.

Teacher–Student Relations (KPI): Positive teacher-student relationships significantly improve performance for both genders, with female students benefiting more strongly from supportive classroom dynamics.

Gender Representation: PISA participation was nearly balanced, with 50.5% female and 49.5% male students, primarily from Grades 10 (58%) and 9 (34%).

Analytical & Predictive Methodology

  • YoY Performance Indexing: Computed club-level YoY change scores (ranging from -1.3 to +0.95) to identify long-term decline and sustained improvement across leagues.
  • Match Outcome Aggregation: Analyzed 25,000+ matches to examine win rates, points accumulation, and competitive balance over multiple seasons.

  • Trend Forecasting: Applied Tableau’s built-in exponential smoothing models to assess momentum and future performance trajectories.

Data Structure

Source: Kaggle – Ultimate Soccer Database (SQLite).
Composition: 7 interrelated tables including Country (11 nations), League, Match, Player, and Team Attributes.
Scale: 25,000+ matches and 10,000+ players across Europe’s top divisions.

Multi-Source Ingestion: API & Unstructured Data

Engineering Architecture

This project implements a robust end-to-end ETL pipeline to ingest, transform, and consolidate fragmented Twitter data into a single, high-integrity analytical dataset. Focusing on the @dog_rates (WeRateDogs) account, the pipeline synchronizes data from live API streams, cloud-hosted flat files, and archival datasets—addressing real-world schema-on-read and data consistency challenges.

Data Pipeline & ETL Implementation

  • Multi-Source Ingestion: Combined three heterogeneous streams: archival CSV data (twitter_archive_enhanced.csv), cloud-hosted TSV files (image-predictions.tsv), and live Twitter metadata queried via Tweepy (REST API).
  • API Engineering: Designed a resilient Tweepy-based extractor to handle rate limiting while retrieving retweet and favorite counts, initially persisted as raw tweet_json.txt.
  • Transformation Engine: Used Pandas, NumPy, JSON normalization, and advanced RegEx to parse unstructured tweet text and extract dog names and growth stages.
  • Schema Denormalization: Merged all sources into a unified, cleaned Gold dataset (twitter_archive_master.csv) suitable for analysis and visualization.

Data Quality & Validation Framework

  • Assess–Clean–Test Workflow: Applied a systematic quality framework aligned with MSc-level data engineering standards.
  • Quality Dimensions: Enforced checks for Completeness, Validity, Accuracy, and Consistency across all merged attributes.
  • Anomaly Resolution: Removed retweets, filtered invalid image URLs, and corrected inconsistent rating and naming patterns.

System Insights

3

Data Streams: Successfully synchronized archival files, cloud-hosted data, and live API responses into one analytical model.

139

Top Breed Insight: Golden Retriever emerged as the most frequent dog breed across valid image predictions.

Engagement Pattern: Strong linear correlation observed between retweet counts and favorite counts across all dog stages.

E-Commerce Recommendation Engine

Problem & System Design

This project addresses the product discovery challenge in large-scale e-commerce platforms. Using the Olist Brazilian E-Commerce dataset (100,000+ orders), I designed a recommendation engine that learns from historical purchase behavior to surface the most relevant products for each customer. The solution focuses on scalable personalization rather than rule-based filtering.

Data Engineering & Feature Pipeline

  • Multi-Table ETL: Integrated 9 relational datasets (customers, orders, products, payments, reviews, sellers) using consistent primary and foreign keys.
  • Data Integrity Validation: Built a custom anomaly detection process to flag logical inconsistencies (e.g. deliveries occurring before order approval).
  • High-Quality Modeling Set: Reduced ~99K raw orders to 95,088 validated transactions suitable for ML modeling.
  • Customer Persistence: Leveraged customer_unique_id to track repeat behavior across multiple order sessions.

Recommendation Logic

  • Hybrid Collaborative Filtering: Combined user–item interaction history with product attributes to infer preference patterns.
  • Encoding Strategy: Applied label encoding for high-cardinality user and product identifiers.
  • Feature Scaling: Normalized price-related features using MinMaxScaler to stabilize similarity scoring.
  • Ranking Engine: Scored all candidate items per user and returned a ranked Top-N recommendation list based on affinity.

Key Outcomes

95K+

Clean Orders: High-integrity transaction set used for recommendation modeling after validation.

9

Unified Tables: End-to-end ETL across customer, product, and transaction domains.

Top-N

Personalization: Ranked product recommendations generated per customer based on behavioral similarity.
